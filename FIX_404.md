# 🔧 FIX RÁPIDO - Erro 404 em todas as rotas

## Problema: Backend retorna 404

### Causa Provável:

O Next.js pode estar em cache antigo ou não reconhecendo as rotas.

---

## ✅ SOLUÇÃO PASSO A PASSO:

### 1️⃣ Pare o servidor

```bash
Ctrl + C
```

### 2️⃣ Limpe o cache do Next.js

```bash
# Windows
rmdir /s /q .next
del /q /s .turbo

# Linux/Mac
rm -rf .next .turbo
```

### 3️⃣ Teste rota simples

Criamos uma rota de health check em: `src/app/api/health/route.ts`

### 4️⃣ Inicie o servidor novamente

```bash
npm run dev
```

### 5️⃣ Teste a rota de health

Abra no navegador:

```
http://localhost:3000/api/health
```

**Esperado:**

```json
{
  "status": "ok",
  "message": "Backend está funcionando!",
  "timestamp": "2024-..."
}
```

---

## Se a rota /api/health funcionar:

Significa que o problema está nos imports (Prisma, bcrypt, etc).

**Solução:**

1. **Instale TODAS as dependências:**

```bash
npm install @prisma/client prisma bcrypt jsonwebtoken zod dotenv @types/bcrypt @types/jsonwebtoken ts-node
```

2. **Gere o cliente Prisma:**

```bash
npx prisma generate
```

3. **Teste novamente:**

```bash
npm run dev
```

---

## Se /api/health TAMBÉM retornar 404:

O Next.js não está reconhecendo as rotas. Possíveis causas:

### A. Versão do Next.js incompatível

```bash
npm install next@latest
```

### B. TypeScript não compilando

```bash
npx tsc --noEmit
```

Se houver erros, corrija-os.

### C. Estrutura de pastas errada

Verifique se os arquivos estão exatamente assim:

```
src/app/api/health/route.ts  ← CORRETO
src/app/api/health.ts         ← ERRADO
src/app/api/healthRoute.ts    ← ERRADO
```

### D. Problema no next.config.js

Temporariamente, use config mínimo:

```javascript
// next.config.js
/** @type {import('next').NextConfig} */
const nextConfig = {
  reactStrictMode: true,
};

module.exports = nextConfig;
```

import { NextResponse } from "next/server";

→ Há algum problema de configuração no seu projeto

Se NÃO funcionar nem no novo:
→ Problema na instalação do Node.js/Next.js
